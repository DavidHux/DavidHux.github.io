集群配置

集群共有8个节点，包括host00的主节点，和host[6-13](不包括7）七个slave节点。暂时

web页面的查看

原本云集群的web管理界面提供了内部节点端口对外映射的功能，但这个功能好像是坏的，只有对于ssh登陆端口的映射是好的。我们使用ssh的本身自带的端口映射功能进行代替。

例如，我们要查看Hadoop的web页面，该页面在Host00的50030端口。可以通过两种方式进行映射。

1. 单对单的映射

可以把Host00的50030端口单独映射到本地机器的一个端口，假设是50031，则在终端中执行以下命令：

之后在浏览器中输入localhost:50031即可查看到hadoop的jobtracker页面。这种方式只代理一个端口，如果要查看多个页面使用多个端口时，可能要执行多条命令进行多个代理。

2. 使用SOCKS进行全局代理

在终端中执行命令

然后在浏览器的代理设置中使用socks代理，设置主机为127.0.0.1，端口为8080（或者设为其他可用的端口）。这种方式下浏览器的web连接都会经过代理，适合于要查看集群主机上多个端口的情况。

相关web ui
